# UTS — Web Client Development

## Deskripsi Tugas
Tugas UTS ini meminta Anda untuk membuat sebuah website tentang hobi atau topik yang diminati, serta melengkapinya dengan laporan singkat berbentuk PDF *slides*.

## Bagian 1: Website
### Ketentuan Umum
* Topik website dibebaskan, namun sangat disarankan mengenai hobi pribadi.
* Website pada dasarnya dapat dibuat menggunakan HTML dan CSS, tetapi tidak ada larangan untuk menggunakan teknologi atau *tools* yang lebih *advanced* (seperti *framework* atau *preprocessor*) selama hak kepemilikan kode (ownership) tetap milik Anda.

### Persyaratan Teknis
**HTML:**
* Diwajibkan menggunakan *tag formatting* teks, seperti `<b>` atau `<strong>`, `<i>` atau `<em>`, `<u>`, dan `<mark>`.
* Harus terdapat minimal satu buah tabel.
* Wajib memiliki minimal 3 menu navigasi. Menu ini tidak wajib berupa 3 file HTML yang terpisah, namun harus ada 3 item menu yang secara visual terlihat pada website.

**CSS:**
* Wajib menggunakan CSS eksternal (menggunakan file `.css` terpisah yang di-link dari dokumen HTML).
* Pewarnaan website harus bagus dan konsisten; disarankan menggunakan alat pembuat *color palette* seperti Coolors atau Adobe Color.
* Website wajib responsif; saat jendela browser dipersempit atau di-*resize*, tampilan layout harus tetap terbaca dengan baik dan tidak berantakan. Penggunaan *media queries* tidak diwajibkan secara ketat asalkan hasil *resize* tetap terlihat rapi.

### Pengumpulan Website
* *Source code* website harus di-*push* ke repositori GitHub.
* Repositori boleh diatur sebagai *public* maupun *private*. Jika repositori diatur menjadi *private*, Anda wajib mengundang **drestanto** sebagai *collaborator*.
* Repositori harus dijaga agar tetap dapat diakses minimal sampai 1 minggu setelah nilai mata kuliah ini keluar.
* Tautan (link) repositori dikumpulkan melalui Google Form berikut: `https://forms.gle/aDziNRZxWjVepCJWA`.
* **Catatan:** Berbeda dengan tugas kecil, repositori ini murni digunakan sebagai media pengumpulan dan tidak akan dinilai secara langsung.

## Bagian 2: Laporan PDF Slides
Selain kode, Anda wajib membuat laporan presentasi yang diekspor ke dalam format PDF, bukan berupa dokumen teks konvensional.

### Ketentuan Laporan
* Format *output* akhir dokumen harus berupa PDF.
* Anda dibebaskan untuk menggunakan *tools* pembuat presentasi apapun (seperti PowerPoint, Canva, atau Google Slides).
* Laporan harus bersifat *engaging* atau menarik secara visual; gunakan *screenshot* tampilan website dan hindari presentasi yang hanya berisi teks.
* Laporan ini dikumpulkan menggunakan Google Form yang sama dengan pengumpulan link repositori GitHub.

### Struktur Slide Laporan
1. **Judul & Topik (Maksimal 1 Slide):** Menjelaskan website tersebut berisi tentang apa dan alasan mengapa topik tersebut dipilih.
2. **Menu & Fitur (Jumlah Slide Bebas):** Menampilkan *screenshot* dari setiap halaman atau menu website beserta penjelasan mengenai fitur-fiturnya.
3. **Link Penting (Maksimal 1 Slide):** Berisi link demo website (misalnya via GitHub Pages atau layanan hosting lain jika ada) serta tautan repositori GitHub.
4. **Kesimpulan (Maksimal 1 Slide):** Menjelaskan poin-poin pelajaran yang didapat selama pengerjaan, tantangan yang dihadapi, serta hal-hal yang masih ingin dikembangkan ke depannya.

## Aspek Penilaian
Website dan laporan PDF dinilai secara komprehensif sebagai hasil akhir, dengan poin-poin sebagai berikut:
* Kelengkapan elemen dasar HTML (seperti keberadaan *formatting*, tabel, dan navigasi).
* Kualitas implementasi CSS yang meliputi pewarnaan, kerapian layout, serta konsistensi visual.
* Responsivitas website untuk memastikan tampilan tetap bagus ketika di-*resize*.
* Kelengkapan serta kualitas penyajian dari laporan PDF slides.
* Tingkat kreativitas dan orisinalitas keseluruhan proyek.
* Penggunaan teknologi *advanced* dipersilakan, tidak akan mengurangi nilai, karena fokus utama penilaian tetap pada kualitas hasil akhir.

Terdapat beberapa topik yang sangat dilarang dan akan mengakibatkan **diskualifikasi** jika digunakan. Topik tersebut meliputi:
